## Descripción

Dado que el MFE consume el API de producto cuando la respuesta es exitosa entonces muestra coberturas y costo y persiste la cotización en Redux durante la sesión

## Pasos

### Paso 1

**Acción:** Configurar el backend/mock del contrato de API para responder exitosamente con el producto, coberturas y costo.

**Resultado esperado:** El endpoint responde 200 con el payload conforme al contrato.

### Paso 2

**Acción:** Cargar la pantalla de oferta del seguro.

**Resultado esperado:** El MFE realiza la llamada al API según el contrato (método, ruta, headers) y no presenta errores.

### Paso 3

**Acción:** Verificar que la UI refleja los datos del API.

**Resultado esperado:** Se muestran coberturas y el costo esperado (incluye $4.99 mensual cuando aplique en el dataset).

### Paso 4

**Acción:** Inspeccionar el estado Redux del MFE (o interfaz de test) tras cargar la pantalla y/o tras seleccionar la cotización.

**Resultado esperado:** La cotización/producto quedan persistidos en el store durante la sesión y son accesibles para el host (según el mecanismo de integración definido).
