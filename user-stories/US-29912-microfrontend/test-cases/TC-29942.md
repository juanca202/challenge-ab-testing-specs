## Descripción

Dado que el API de producto falla cuando cargo la pantalla de oferta entonces el MFE muestra manejo de error sin romper el host ni perder el marco de la aplicación

## Pasos

### Paso 1

**Acción:** Configurar el API/mock para responder con error (p. ej. 500) o timeout.

**Resultado esperado:** El endpoint devuelve error o no responde en el tiempo configurado.

### Paso 2

**Acción:** Navegar a la pantalla de oferta del seguro.

**Resultado esperado:** El MFE intenta consumir el API y detecta el fallo.

### Paso 3

**Acción:** Verificar el comportamiento ante error en la UI.

**Resultado esperado:** Se muestra un mensaje de error/estado vacío controlado (sin pantalla en blanco), y el usuario puede continuar con acciones permitidas (p. ej. reintentar o continuar onboarding según diseño).

### Paso 4

**Acción:** Verificar estabilidad del host.

**Resultado esperado:** El header y navegación del onboarding siguen operativos; no hay errores no controlados en consola que rompan la app.
